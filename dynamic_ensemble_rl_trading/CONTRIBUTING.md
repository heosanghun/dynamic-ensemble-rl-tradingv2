# Contributing Guidelines

This repository is prepared for anonymous paper submission. Contributions should maintain anonymity.

## Code Style

- Follow PEP 8 style guidelines
- Use type hints for function parameters and return values
- Include docstrings for all classes and functions
- Keep comments professional and clear

## Testing

- Write unit tests for new features
- Ensure all tests pass before submitting
- Maintain test coverage above 80%

## Documentation

- Update relevant documentation when adding features
- Keep README.md up to date
- Document any breaking changes

## Anonymity

- Do not include personal information
- Do not use author names or emails
- Keep all repository links anonymous

## Pull Request Process

1. Create a feature branch
2. Make your changes
3. Ensure tests pass
4. Update documentation
5. Submit pull request

Note: For anonymous submissions, use the provided anonymous repository link format.
